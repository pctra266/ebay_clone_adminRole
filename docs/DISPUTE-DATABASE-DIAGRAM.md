## ?? DISPUTE LIFECYCLE FLOW

```
??????????????????????????????????????????????????????????????????
?                    COMPLETE WORKFLOW                            ?
??????????????????????????????????????????????????????????????????

1. ORDER COMPLETED
   ?? Order delivered
   ?? CanDisputeUntil = DeliveredAt + 30 days
   ?? Buyer can open dispute
        ?
        ?
2. BUYER OPENS CASE
   ?? Status: Open
   ?? Type: INR/INAD/Damaged/Counterfeit
   ?? Upload evidence (BuyerEvidence)
   ?? Set DesiredOutcome
   ?? System actions:
   ?  ?? Generate CaseId (DSP-20240131-001)
   ?  ?? Set Deadline (72h for seller response)
   ?  ?? Move money: AvailableBalance ? DisputedBalance
   ?  ?? Send notification to seller
   ?? Status ? AwaitingSellerResponse
        ?
        ?
3. SELLER RESPONDS (within 72h)
   ?? Option A: Accept Return
   ?  ?? Status ? Resolved (Buyer Wins)
   ?
   ?? Option B: Offer Partial Refund
   ?  ?? Create DisputeMessage (type: Offer)
   ?  ?? Status ? Negotiating
   ?  ?? Wait for buyer response
   ?     ?? Buyer accepts ? Resolved
   ?     ?? Buyer declines ? Continue
   ?
   ?? Option C: Deny with Evidence
      ?? Upload SellerEvidence
      ?? Status ? Negotiating
      ?? Wait for buyer counter-response
           ?
           ?
4. NEGOTIATION (Up to 3 rounds)
   ?? Back-and-forth offers
   ?? NegotiationRounds++
   ?? Store in OfferHistory
   ?? If agreement ? Resolved
   ?? If no agreement ? Escalated
        ?
        ?
5. ESCALATED TO ADMIN
   ?? Status: Escalated
   ?? EscalatedAt timestamp
   ?? Calculate Priority (amount + time + user profile)
   ?? Add to admin queue
   ?? Send notification to admin
        ?
        ?
6. ADMIN ASSIGNMENT
   ?? Admin clicks "Assign to Me"
   ?? Status: AssignedToAdmin
   ?? AssignedTo = AdminId
   ?? AssignedAt timestamp
   ?? Status ? UnderReview
        ?
        ?
7. ADMIN REVIEW
   ?? View TAB 1: Buyer's Claim
   ?? View TAB 2: Seller's Response
   ?? View TAB 3: Chat History
   ?? View TAB 4: System Timeline
   ?? View TAB 5: Admin Analysis
   ?? ViewCount++
   ?? LastViewedAt updated
        ?
        ?
8. ADMIN DECISION
   ?? Option A: Buyer Wins
   ?  ?? Winner = 'Buyer'
   ?  ?? RefundAmount = Full amount
   ?  ?? seller.ViolationCount++
   ?  ?? Process full refund
   ?
   ?? Option B: Seller Wins
   ?  ?? Winner = 'Seller'
   ?  ?? RefundAmount = 0
   ?  ?? Move DisputedBalance ? AvailableBalance
   ?  ?? Close case
   ?
   ?? Option C: Split Decision
      ?? Winner = 'Split'
      ?? RefundAmount = Partial
      ?? Process partial refund
      ?? Both parties compromised
           ?
           ?
9. REFUND PROCESSING
   ?? Create FinancialTransaction (Refund to Buyer)
   ?? Create FinancialTransaction (Deduction from Seller)
   ?? Update SellerWallet:
   ?  ?? DisputedBalance -= Amount
   ?  ?? TotalRefunded += RefundAmount
   ?  ?? AvailableBalance -= (Amount - RefundAmount)
   ?? Update Order.Status
   ?? RefundProcessedAt timestamp
   ?? RefundTransactionId from gateway
        ?
        ?
10. NOTIFICATIONS & CLOSURE
    ?? Send email to buyer (decision + refund info)
    ?? Send email to seller (decision + impact)
    ?? Create DisputeMessage (SystemUpdate)
    ?? Log AdminAction
    ?? Status ? Resolved
    ?? After refund complete: Status ? Closed
```
